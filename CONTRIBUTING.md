## Contributing

Thank you for your interest in contributing to this project.

To help keep the codebase clean, maintainable, and consistent, please follow the guidelines below before submitting your contribution.

### Development Workflow

1. Fork the repository (if applicable).
2. Create a new branch from the latest `main` branch.

```bash
git checkout -b feature/your-feature-name
```

3. Implement your changes following the project's coding standards.
4. Test your changes locally to ensure everything works as expected.
5. Commit your changes using clear and descriptive commit messages.

```bash
git commit -m "feat: add user session validation"
```

6. Push your branch.

```bash
git push origin feature/your-feature-name
```

7. Open a Pull Request with a clear description of your changes.

### Coding Standards

- Follow the Laravel framework conventions.
- Write clean, readable, and maintainable code.
- Keep functions and classes focused on a single responsibility.
- Use meaningful variable, method, and class names.
- Remove unused code before submitting.
- Ensure there are no linting or formatting issues.
- Update documentation when introducing new features or significant changes.

### Pull Request Checklist

Before opening a Pull Request, make sure that:

- Your branch is up to date with the latest `main` branch.
- The application builds successfully.
- All tests pass.
- New functionality has been tested.
- Existing functionality has not been broken.
- Documentation has been updated if required.

We appreciate every contribution that helps improve this project.
